Nexus Documentation

Nexus SaaS Admin Dashboard

Premium admin dashboard template built with Next.js, React, TypeScript, and Tailwind CSS for SaaS products, analytics platforms, billing workflows, and internal operations tools.

What This Template Is

Nexus is a multi-page SaaS admin template with polished dark surfaces, reusable components, interactive table states, analytics visuals, auth flows, and responsive layouts. It is designed to be used as a starter UI for dashboards, back-office systems, analytics products, CRM tools, billing consoles, and internal admin apps.

Installation

npm install
npm run dev
npm run build

For local development, open http://localhost:3000 after starting the dev server. For production preview, run npm run start after the build completes.

How To Change Branding And Colors

Update brand and product values in src/config/product.ts. This controls shared product name, company name, support labels, workspace identity, demo user details, and related brand text.

Update color tokens and theme styling in src/app/globals.css and tailwind.config.ts.

How To Replace Mock Data With A Real API

  1. Identify the route you want to connect, for example src/app/users/page.tsx.
  2. Locate its shared demo source in src/data/, for example src/data/users.ts.
  3. Replace the imported mock arrays with data fetched from your API or server actions.
  4. Keep the shared TypeScript shapes in src/types/ and map your API response into those models.
  5. Move hardcoded UI state messages into your real loading, empty, success, and error flows as needed.

The current structure is intentionally prepared for API swapping: page files mostly focus on state and rendering, while the demo records already live in shared data and types modules.
